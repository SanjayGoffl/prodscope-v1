## 🔍 Deep Dive: How it Works Right Now

### **Phase 1: Product Definition**
- **Inputs**: Name, Price, Description, Features.
- **Mechanism**: Captures text inputs and manages an array of "feature chips". 
- **Current Limitation**: UI is basic; missing advanced fields like "Target Audience" or "Category".

### **Phase 2: Competitor Discovery**
- **Mechanism**: Calls `callLLM` (Gemini) with product context to suggest 5 competitors. Allows manual addition.
- **Mechanism**: Stores competitors in the global `state.competitors` array.

### **Phase 3: Extraction Pipeline**
- **Mechanism**: A visual sequence using `setTimeout` (delay) to mimic complex backend processing.
- **AI Task**: Calls LLM (Reasoning model) to generate a structured JSON matrix comparing features across all selected products.

### **Phase 4: Comparison Matrix**
- **Mechanism**: Dinamically builds a large HTML table with color-coded "Yes/No/Partial" cells.
- **Charts**: Renders feature availability bars and category coverage charts.

### **Phase 5: Results & Export**
- **Mechanism**: Calculates market position metrics (Relative pricing, Feature parity).
- **Export**: Generates a `.txt` formatted report based on the current analysis.

### **Phase 6: Persona Simulation**
- **Mechanism**: Iterates through 4 pre-defined synthetic personas (Budget Buyer, Luxury Buyer, etc.).
- **LLM Choice**: For each persona, a prompt is sent to `groq_fast` to decide which product they would buy and why.

### **Phase 7: Final Dashboard**
- **Mechanism**: Aggregate of simulation results. Shows market share (Pie) and probability of purchase (Bar) for each competitor.
